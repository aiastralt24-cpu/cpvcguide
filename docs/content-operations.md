# Content Operations Workflow

1. Topic Intake
   - lock the primary query
   - confirm the audience and intent
   - run a similarity and cannibalization check
   - approve, merge, narrow, or reject the topic
2. Research Brief
   - lock the reader problem
   - lock the article angle
   - lock technical concepts, FAQs, and what to avoid
3. Draft
   - answer the query in the first screenful
   - include limits, conditions, and next-step guidance
   - add metadata and related links
4. Editorial Review
   - remove filler and weak intros
   - improve structure, readability, and usefulness
   - validate SEO and AEO readiness
5. Technical Review
   - validate standards, safety, installation, and performance claims
   - decide whether the page is only publishable or also indexable-ready
6. Publish Decision
   - final metadata and rendering check
   - confirm `published` separately from `indexable`
7. Refresh
   - 90-day review
   - improve, consolidate, retire, or upgrade indexability
